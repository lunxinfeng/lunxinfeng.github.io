android framework之旅（三）编译调试单个模块

发表于 2018-04-23 | 分类于 Android

在开发过程中，我们需要不断修改代码，并进行调试，而源码的整体编译耗时太长，我们不可能每次测试的时候都要重新生成镜像，重新烧系统，那我估计人会发疯的，所以这两天研究了下怎么单独编译模块并push到真机看到效果，过程还是比较曲折的，记录一下～

环境

操作系统：ubuntu 16.04
开发板：NanoPC T3
源码版本：5.1.1

编译单个模块

为了验证我们的工作，我在Laucher的onCreate方法中加了这么一句话：

System.out.println(“test tag add by lxf”);

首先还是初始化环境和选择目标版本：

source build/envsetup.sh
lunch <自己要编译的版本>

执行初始化之后在当前目录我们可以得到一些其他指令，比较常用的就是mm和mmm指令（其实是我只用过这两个），mm表示编译当前目录，mmm表示编译指定目录，只要目录下又Android.mk文件即可。
比如我们编译桌面模块（我使用开发板的桌面为Launcher2，有的为Launcher3）：

mmm packages/apps/Launcher2/

编译过程中会提示你具体的生成目录：

android framework之旅（一）下载源码

发表于 2018-04-09 | 分类于 Android

公司产品需要和硬件通信，最近有需求要去掉每次USB插入时的弹窗授权提示，研究了两天最后发现最完美的解决方法居然是修改android系统源码，So～开始我的入坑之旅。

准备工作

Android源码的编译目前是只支持Mac和Linux的，考虑到编译时的性能问题，我没有使用虚拟机，而是装了win10和ubuntu双系统，安装方法可以自行百度，近乎傻瓜式操作。
源码的下载要用到git工具，所以还要先安装git：

sudo apt-get install git

配置用户名和邮箱，不配置的话下载源码的时候也会提醒你进行配置：

git config –global user.name “your name”
git config –global user.email “you email”

安装curl（一个http命令行工具），用于下载repo工具：

sudo apt-get install curl

下载源码

android源码较大，在国内Google被墙的情况下，建议使用国内镜像下载，当然你有科学的上网姿势也可以～～镜像的话有清华镜像和中科大镜像，这里我使用的是清华镜像。
网络上关于使用清华镜像下载android源码的教程很多，但是很多比较老，有很多坑，比如清华镜像站在16年就已经停用了git协议，只支持https，别问我怎么知道的/(ㄒoㄒ)/~~，建议直接查看清华镜像站的官方使用帮助。

View系列笔记（二）事件分发机制

发表于 2018-03-21 | 分类于 Android

本文基于Android SDK 26。

我们知道，android的view是树形结构，当我们点击一个位置时，会有多个view可以作出响应，那此时这个点击事件应该交给谁处理，这就需要用到事件分发机制。

ViewGroup事件分发

在了解事件分发机制之前，我们先来声明一个概念：事件序列。

同一个事件序列是指从手指接触屏幕的一刻起，到手指离开屏幕的那一刻结束，这个过程中所产生的一系列事件。包括一个down事件+n个move事件+一个up事件。

所谓事件分发，其实就是产生一个MotionEvent事件产生后，Android系统将这个事件在层层View之间传递和处理的过程。主要涉及到三个方法：

  • boolean dispatchTouchEvent(MotionEvent)：用来进行事件的分发，如果事件能够传递到该view，则该方法一定会被触发，返回结果受该view的onTouchEvent()和下个view的dispatchTouchEvent()影响，表示是否消耗该事件。
  • boolean onInterceptTouchEvent(MotionEvent)：用来表示是否拦截这个事件。如果拦截，则在这个事件序列中不会再调用该方法。注意此方法是ViewGroup特有的。
  • boolean onTouchEvent(MotionEvent)：用来处理点击事件，返回结果表示是否消耗当前事件。如果不消耗，在同一个事件序列中，该view无法再次收到事件。

Kotlin学习笔记（四）约定(一)

发表于 2018-02-24 | 分类于 Kotlin

约定，指使用与常规方法调用语法不同的、更简洁的符号，调用有着特殊命名的函数。举个例子：如果你在类中定义了一个名为plus的特殊方法（需要用operator关键字来声明该方法），那么按照约定，你就可以在该类的实例上使用+运算符。

算术运算符约定

二元算术运算符

表达式 函数名
a + b plus
a - b minus
a * b times
a / b div
a % b rem

我们使用运算符时，实际上调用的是与之对应的函数。 注意事项：

  • 这种自定义的运算符，基本与标准数字类型的运算符具有相同的优先级，a+b*c时会先执行乘法（times函数）。
  • 不支持交换性，即a+b和b+a是不同的，如果需要支持交换性，可以定义一个额外的运算符函数。
